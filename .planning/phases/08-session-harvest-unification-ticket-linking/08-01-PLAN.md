# Phase 8 Plan 01: Pluggable Refactor + Unified Collection

**Phase:** 08-session-harvest-unification-ticket-linking
**Plan:** 01
**Wave:** 1
**Depends on:** (none — first plan in phase)
**Autonomous:** true
**Requirements:** REQ-CI-05

---

## Goal

Introduce the `SessionSource` adapter interface and normalized session/message model, refactor the existing `OpenCodeSQLiteHarvester` and `ClaudeCodeHarvester` into thin adapters behind a generic harvest engine, unify all session documents into the `sessions` collection (migrate existing `session-summary` docs), and add a per-source `case` in `buildSourcePath`. No behavior loss; `memory_wake_up` collection counts become correct.

---

## Success Criteria

1. `SessionSource` interface compiled and documented in `internal/harvest/source.go`
2. `NormalizedSession` and `NormalizedMessage` types defined in `internal/harvest/model.go`
3. `GenericHarvestEngine` drives discover → read → normalize → dedup → skip-active → render → summarize/persist → raw fallback; existing tests green
4. OpenCode SQLite and Claude Code harvesters thin-wrap their adapters; no duplicate discovery/persist logic
5. `persist.go:buildSourcePath` has an explicit `case` for each known source (`claude`, `opencode`); unknown sources return `summary://<source>/`
6. `Persister.Save` writes to collection `sessions` (not `session-summary`)
7. A migration function migrates all `session-summary` docs to collection `sessions` (idempotent SQL UPDATE)
8. `wakeup.go` collection filter updated from `["memory","session-summary"]` to `["memory","sessions"]`
9. `go test -race -short ./...` passes

## Tasks

### Task 1: Define `SessionSource` interface and normalized model

**Files:**
- `internal/harvest/source.go` (new)
- `internal/harvest/model.go` (new)

**Steps:**

1. Create `internal/harvest/source.go`:
   - Define `Location` struct: `DBPath string`, `SessionDir string`, `WorkspaceHash string`, `WorktreePath string`
   - Define `SessionSource` interface with three methods:
     - `Name() string` — returns the source identifier (e.g. `"opencode"`, `"claude"`)
     - `Discover(ctx context.Context, registered map[string]string) ([]Location, error)` — scans for matching session stores; `registered` is path→hash
     - `Read(ctx context.Context, loc Location) ([]NormalizedSession, error)` — reads and parses all sessions from one location
   - Add compile-time check stubs (leave unimplemented adapters as placeholders with `//nolint:unused`)

2. Create `internal/harvest/model.go`:
   - Define `NormalizedMessage` struct: `Role string`, `Content string`, `Timestamp time.Time`, `ToolName string`, `IsSidechain bool`
   - Define `NormalizedSession` struct: `Source string`, `SessionID string`, `ParentID string`, `WorkspaceHash string`, `Branch string`, `Cwd string`, `Title string`, `CreatedAt time.Time`, `UpdatedAt time.Time`, `Messages []NormalizedMessage`
   - Add `IsActive() bool` method on `NormalizedSession`: returns `!UpdatedAt.IsZero() && time.Since(UpdatedAt) < 10*time.Minute` (mirrors existing `isActiveSession` logic at `opencode_sqlite.go:274`)

**Acceptance criteria:**
- Both files compile in `package harvest`
- `SessionSource` interface is exported and documented
- `NormalizedSession.IsActive()` matches existing `isActiveSession` behavior exactly

### Task 2: Implement adapters + generic engine

**Files:**
- `internal/harvest/opencode_source.go` (new)
- `internal/harvest/claude_source.go` (new)
- `internal/harvest/engine.go` (new)

**Steps:**

1. Create `internal/harvest/opencode_source.go` — `OpenCodeSource` struct implementing `SessionSource`:
   - Constructor `NewOpenCodeSource(root string, logger zerolog.Logger) *OpenCodeSource` where `root` is the OpenCode DB root directory (the parent of per-project dirs that `ScanOpenCodeDBRoot` scans at `opencode_sqlite.go:577`)
   - `Name()` returns `"opencode"`
   - `Discover` delegates to existing `ScanOpenCodeDBRoot` (`opencode_sqlite.go:572-638`), converts `[]DiscoveredDB` to `[]Location`
   - `Read` opens the SQLite DB at `loc.DBPath`, calls the existing unexported `listSessions` + `listMessages` logic (extract into package-level helpers if needed, keeping the originals intact), and maps results to `[]NormalizedSession`. Preserve `ParentID` from the SQLite `session.parent_id` column (add it to the `listSessions` SELECT — it is already indexed per RESEARCH.md)

2. Create `internal/harvest/claude_source.go` — `ClaudeSource` struct implementing `SessionSource`:
   - Constructor `NewClaudeSource(sessionDir string, logger zerolog.Logger) *ClaudeSource`
   - `Name()` returns `"claude"`
   - `Discover` scans `sessionDir` for `*.jsonl` files, returns one `Location` per file (set `SessionDir` to the file path, `WorkspaceHash` is resolved by the caller)
   - `Read` calls existing `parseJSONLFile` (`claudecode.go:268`) and maps `[]claudeCodeMessage` to `[]NormalizedSession`. Populate `Branch` and `Cwd` from per-record `gitBranch` and `cwd` JSON fields (add them to `claudeCodeMessage` struct — they are present in Claude JSONL but not currently parsed)

3. Create `internal/harvest/engine.go` — `Engine` struct:
   - Constructor `NewEngine(source SessionSource, pgDB *sql.DB, summarizer SessionSummarizer, enqueuer ChunkEnqueuer, logger zerolog.Logger) *Engine`
   - `HarvestAll(ctx context.Context, enqueuer ChunkEnqueuer) (harvested, skipped, errCount int)` — implements the existing `Harvester` interface so it can be added to `Runner` unchanged
   - Pipeline: `source.Discover` → for each location: `source.Read` → for each `NormalizedSession`: `IsActive()` skip → content-hash dedup via `GetDocumentBySourcePath` → render markdown (extract `renderSQLiteMarkdown`/`renderClaudeCodeMarkdown` logic into a `RenderMarkdown(sess NormalizedSession) string` helper in `model.go`) → `summarizer.SummarizeAndPersist` or `writeRawFallback`
   - `writeRawFallback` in engine writes to `Collection: "sessions"` (not `"session-summary"`)
   - Source path uses `"summary://<source>/<sessionID>"` — same pattern as today, works with updated `buildSourcePath`

**Acceptance criteria:**
- Both adapter structs satisfy `SessionSource` at compile time (add `var _ SessionSource = (*OpenCodeSource)(nil)` and `var _ SessionSource = (*ClaudeSource)(nil)`)
- `Engine` satisfies `Harvester` at compile time: `var _ Harvester = (*Engine)(nil)`
- `go build ./internal/harvest/...` passes
- Existing `OpenCodeSQLiteHarvester` and `ClaudeCodeHarvester` structs are NOT deleted yet (kept to avoid breaking wiring in `main.go`; wiring update is in Task 4)

### Task 3: Unify collection + migrate `session-summary` docs

**Files:**
- `internal/summarize/persist.go`
- `internal/server/handlers/wakeup.go`
- `internal/harvest/migration.go` (new)

**Steps:**

1. In `internal/summarize/persist.go`:
   - Change line 119: `Collection: "session-summary"` → `Collection: "sessions"` in `UpsertDocumentBySourcePathParams`
   - Change line 165 (link extractor call): update `Collection: "session-summary"` → `Collection: "sessions"` in the `links.Document` passed to `linkExtractor.Extract`
   - Update `buildSourcePath` (lines 211-218): add explicit `case "opencode": return "summary://opencode/" + meta.SessionID` before the default; keep `case SourceClaude`; change `default` to `return "summary://" + string(meta.Source) + "/" + meta.SessionID` (handles future sources without wrong fallback)

2. Create `internal/harvest/migration.go` — `MigrateSessionSummaryToSessions(ctx context.Context, db *sql.DB) (int64, error)`:
   - Runs a single SQL UPDATE: `UPDATE documents SET collection = 'sessions' WHERE collection = 'session-summary'`
   - Returns `rowsAffected, nil`
   - Idempotent: if 0 rows match, returns `0, nil` with no error
   - Logs the count at Info level
   - Does NOT move chunks (they are keyed by document_id, not collection; no migration needed)

3. In `internal/server/handlers/wakeup.go` line 91:
   - Change `Collections: []string{"memory", "session-summary"}` → `Collections: []string{"memory", "sessions"}`

**Acceptance criteria:**
- `Persister.Save` writes `collection = "sessions"` for all new summaries
- `MigrateSessionSummaryToSessions` is callable and idempotent (run twice → second call returns 0 affected)
- `WakeUpHandler` queries `memory` + `sessions` collections
- `go test -race -short ./internal/summarize/... ./internal/server/...` passes

## Risks

| Risk | Mitigation |
|------|------------|
| Migration races with an in-flight harvest writing `session-summary` | Migration runs before Runner starts; Runner startup is serial |
| Wakeup tests hardcode `"session-summary"` | Update test fixtures alongside `wakeup.go` change |
| `buildSourcePath` default case changes behavior for unknown sources | New default is `"summary://<source>/<id>"` — more correct, not less |
| Engine flag path never toggled | Flag defaults `false`; can be removed once confidence is established in Phase 8 follow-up |
