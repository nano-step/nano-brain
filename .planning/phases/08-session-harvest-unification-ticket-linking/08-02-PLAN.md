## Goal

Thread `Branch`, `Cwd`, and `ParentID` through `SummaryMeta` → `SessionMetadata` → front-matter. Implement a ticket extractor that derives ticket IDs from message content, branch name, and parent-session inheritance. Store ticket IDs as `ticket:<ID>` tags on the persisted document. Wire the latent `RelationshipLookup` in `pipeline.go` so parent/child sessions populate the summary header. Make ticket regex patterns configurable.

---

1. `SummaryMeta` (in `internal/harvest/harvest.go`) has `Branch string`, `Cwd string` fields populated by both adapters
2. `SessionMetadata` (in `internal/summarize/pipeline.go`) has `Branch string`, `Cwd string` fields; `formatHeader` emits them when non-empty
3. `TicketExtractor` in `internal/harvest/tickets.go` extracts `[A-Z]+-\d+` and `#\d+` patterns from content + branch; parent-session tickets inherited if `ParentID` is set
4. Extracted tickets added as `ticket:<ID>` tags in both `Persister.Save` (summarized path) and `writeRawFallback` (raw path)
5. `RelationshipLookup` wired with a real implementation that queries `documents` by `source_path` to resolve parent title + children; `NewPipeline` at `main.go:796` receives it instead of `nil`
6. Ticket regex patterns configurable via `config.yml` under `harvest.ticket_patterns` (slice of regex strings; default `["[A-Z]+-\\d+", "#\\d+"]`)

### Task 2: Implement `TicketExtractor` with configurable patterns + parent inheritance

**Files:**
- `internal/harvest/tickets.go` (new)
- `internal/harvest/tickets_test.go` (new)
- `internal/config/config.go` (add `TicketPatterns []string` to harvest config)

**Steps:**

1. Create `internal/harvest/tickets.go`:

   - Define `TicketExtractor` struct: `patterns []*regexp.Regexp`

   - Constructor `NewTicketExtractor(patterns []string) (*TicketExtractor, error)`:
     - Compile each pattern; return error if any fails to compile
     - If `patterns` is empty, use default patterns: `[A-Z]+-\d+` and `#\d+`

   - Method `Extract(content, branch, parentTags []string) []string`:
     - Run each compiled regex over `content` and `branch` (deduplicated)
     - For each match, normalize to uppercase (Jira IDs are uppercase by convention)
     - Inherit tickets from `parentTags`: filter `parentTags` for strings with prefix `"ticket:"`, strip the prefix, add to the set
     - Return sorted, deduplicated slice of bare ticket IDs (e.g. `["DEV-4706", "DEV-4707"]`)

   - Method `AsTags(tickets []string) []string`:
     - Returns `["ticket:DEV-4706", "ticket:DEV-4707"]` etc.

2. Create `internal/harvest/tickets_test.go`:
   - `TestExtract_ContentMatch`: content with `DEV-1234` → extracted
   - `TestExtract_BranchMatch`: branch `feat/DEV-4706-feature` → `DEV-4706` extracted
   - `TestExtract_ParentInheritance`: parent has tag `ticket:DEV-999`; child content has no ticket → child inherits `DEV-999`
   - `TestExtract_SubagentNoContent`: empty content, no branch, parent tag `ticket:PROJ-42` → result `["PROJ-42"]`
   - `TestExtract_Deduplicate`: same ticket in content and branch → appears once
   - `TestExtract_CustomPattern`: pattern `#\d+` matches `#42` in content

3. In `internal/config/config.go`, add to the harvest/claude harvester config section:
   - `TicketPatterns []string \`yaml:"ticket_patterns"\`` with doc comment; zero value means use defaults

**Acceptance criteria:**
- `go test -race -short ./internal/harvest/...` passes with all new test cases green
- `TicketExtractor` is exported and usable from `engine.go`

3. Create `internal/summarize/relationship.go` — `DBRelationshipLookup` struct implementing `RelationshipLookup`:

     - If any lookup fails, log warn and return nil (non-fatal — matches existing `pipeline.go:97-100` pattern)

   - Pass `relLookup` instead of `nil` in `summarize.NewPipeline(llmClient, relLookup, ...)` (line 796)

## Risks

| Risk | Mitigation |
|------|------------|
| `#\d+` pattern produces false positives (e.g. markdown headers `#Introduction`) | Default content scan strips markdown headers before applying `#\d+`; branch scan is unaffected |
| Parent tag lookup adds a DB round-trip per session | Single `GetDocumentBySourcePath` call; negligible vs. LLM summarization latency |
| Child lookup via JSON metadata `parent_id` field is fragile (schema-less) | First iteration: children not populated if metadata is missing; `Children` stays empty gracefully (pipeline already handles `nil` children at `pipeline.go:252`) |
| Ticket patterns config zero value | `NewTicketExtractor(nil)` uses built-in defaults; no regression |
