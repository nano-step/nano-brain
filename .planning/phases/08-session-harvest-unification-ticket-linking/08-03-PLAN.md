## Goal

Expose a query path that returns all sessions tagged with a given ticket ID across ALL registered workspaces and sources. Wire it as an MCP tool (`memory_ticket`) so agents can ask "what work has been done on DEV-4706?" and get a unified view across repos and agent types.

---

## Success Criteria

1. A `TicketSessionQuerier` interface and `DBTicketSessionQuerier` implementation exist in `internal/server/handlers/` (or a dedicated `internal/ticket/` sub-package)
2. A new sqlc query `ListDocumentsByTag` (or equivalent) returns documents from all workspaces matching `tag = "ticket:<ID>"` and `collection = "sessions"`
3. HTTP endpoint `GET /api/sessions/by-ticket?ticket=<ID>` returns JSON array of session summaries (title, source, workspace, session_id, tags, snippet)
4. An MCP tool `memory_ticket` is registered and calls the endpoint; returns formatted markdown list
5. The query is cross-workspace: it returns results from ALL registered workspaces (not filtered to the calling workspace)
6. `go test -race -short ./...` passes; handler has a unit test with a mock querier

## Tasks

### Task 1: Add `ListDocumentsByTag` sqlc query + implement querier

**Files:**
- `internal/storage/sqlc/queries/documents.sql` (or whichever `.sql` file contains document queries â€” check existing pattern)
- `internal/storage/sqlc/` (generated; run `sqlc generate` after SQL change)
- `internal/server/handlers/ticket.go` (new)
- `internal/server/handlers/ticket_test.go` (new)

**Steps:**

1. Locate the correct `.sql` file by finding existing document queries (e.g. `ListDocumentsByCollection`). Add a new query:

   ```sql
   -- name: ListDocumentsByTag :many
   SELECT id, workspace_hash, title, content, source_path, collection, tags, metadata, created_at, updated_at
   FROM documents
   WHERE $1 = ANY(tags)
     AND collection = $2
   ORDER BY updated_at DESC
   LIMIT $3;
   ```

   Parameters: `tag text`, `collection text`, `limit int32`. The `$1 = ANY(tags)` pattern matches a single tag value against the PostgreSQL text array column â€” consistent with how existing tag filtering works in the codebase.

2. Run `sqlc generate` to produce the Go method `ListDocumentsByTag`.

3. Create `internal/server/handlers/ticket.go`:

   - Define `TicketQuerier` interface: `ListDocumentsByTag(ctx, arg sqlc.ListDocumentsByTagParams) ([]sqlc.ListDocumentsByTagRow, error)`

   - Define `TicketSessionResult` struct: `SessionID string`, `Title string`, `Source string`, `WorkspaceHash string`, `SourcePath string`, `Tags []string`, `Snippet string` (first 300 chars of content)

   - Implement `TicketHandler(q TicketQuerier, logger zerolog.Logger) echo.HandlerFunc`:
     - Read `ticket` query param; reject empty with 400
     - Normalize: strip leading `#` if present; if the value contains only digits, prepend `#` to search (support both `DEV-4706` and raw `42` for `#42` style)
     - Build tag value: `"ticket:" + normalizedTicket`
     - Call `q.ListDocumentsByTag(ctx, sqlc.ListDocumentsByTagParams{Tag: tagValue, Collection: "sessions", Limit: 50})`
     - Map each row to `TicketSessionResult`; extract `Source` from `tags` (look for tag that matches `"opencode"` or `"claude"`) or from `source_path` prefix
     - Return 200 JSON array

4. Create `internal/server/handlers/ticket_test.go`:
   - Mock `TicketQuerier` returning 2 docs from different workspaces
   - Assert response is 200, JSON array length 2, each has correct `session_id` and `source`
   - Test empty `ticket` param â†’ 400
   - Test querier error â†’ 500

**Acceptance criteria:**
- `sqlc generate` succeeds with no errors
- `go test -race -short ./internal/server/handlers/...` passes including ticket handler tests
- Handler returns results from all workspaces (not filtered by workspace in the SQL query)

## Risks

| Risk | Mitigation |

| Cross-workspace query returns sensitive sessions | No new auth surface â€” existing row-level security (workspace-scoped API keys) does not apply here; document that `memory_ticket` is a privileged tool requiring server-local access |
