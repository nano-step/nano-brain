<objective>
Drop `"workspace"` from the required-fields list of the 14 `memory_*` tools that currently require it, and update each tool's `workspace` parameter description to note it is optional when the connection was configured with a `?workspace=` URL default (D-06). Keep the parameter itself in every schema — only required-ness changes. Do not touch the 4 tools that never required a workspace.

Purpose: Without this, an LLM agent is still schema-forced to supply `workspace` even when the connection binds a default, defeating the whole phase (the ~22% wake_up problem in CONTEXT.md `<specifics>`). The runtime fallback (Plan 01) makes omission WORK; this plan makes omission ALLOWED by the schema.
Output: 14 mechanical schema edits + a schema-assertion test.
</objective>
