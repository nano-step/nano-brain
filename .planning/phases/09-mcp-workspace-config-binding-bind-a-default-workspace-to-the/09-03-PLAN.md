<objective>
Prove the mechanism end-to-end and document it. Add the single most important test in this phase: a full HTTP round-trip against a real Echo instance where `?workspace=` is on the URL and the tool call omits the `workspace` arg — this is the ONLY test that would catch the `echo.MiddlewareFunc`-vs-plain-`http.Handler` wiring mistake (RESEARCH Pitfall 1), because the in-memory-transport helper bypasses HTTP entirely. Extend the write-path security test for D-05, and add the `?workspace=` example to the three config docs.

Purpose: The in-memory tests in Plans 01/02 verify precedence and schema but CANNOT catch a wiring mistake where the middleware never reaches the SDK's `req.Context()`. This plan closes that gap and ships the user-facing config documentation that makes the feature usable.
Output: An integration test, a security-test extension, and three doc edits.
</objective>
