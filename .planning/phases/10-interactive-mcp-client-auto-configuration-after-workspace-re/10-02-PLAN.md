<objective>
Implement the interactive, idempotent, multi-client MCP auto-configuration: after `--root` registration on a TTY, prompt per client and read-modify-write each selected client's project-local (Claude Code, OpenCode) or global (Codex CLI) config, binding nano-brain to `?workspace=<name>`.

Purpose: This is the phase's core deliverable (issue #525). It closes the gap between "Phase 9 unlocked per-project `?workspace=` bindings" and "a user actually gets one written for them" — turning a 3-file, 3-schema hand-edit into per-client Y/N prompts. Depends on Plan 01 for `result.Name`.
Output: A new `mcp_client_config.go` with the prompt orchestrator + JSON/TOML merge helpers + 3 client writers, new detection helpers in `detect.go`, the wiring in `commands.go`, and table-driven tests including a realistic Codex TOML round-trip fixture.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → local filesystem | The CLI writes config files under the project root (`.mcp.json`, `opencode.json`) and the user home (`~/.codex/config.toml`). Path is derived from `result.RootPath`, which is `filepath.Abs`-resolved server-side before the CLI sees it. |
| Existing config file → merge logic | User-authored (possibly hand-edited) JSON/TOML is parsed and re-serialized; unrelated keys/servers must survive untouched (D-05). |
| Workspace name → generated URL | Server-controlled `name` (`filepath.Base`) is interpolated into a URL query string in the written config. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-03 | Tampering | mergeJSONMCPEntry / mergeCodexTOMLEntry silently dropping or corrupting unrelated user keys/servers (D-05 core risk across 3 schemas) | high | mitigate | Whole-file `map[string]any` model (never typed structs) so unknown keys round-trip; set only the single `nano-brain` entry; dedicated preserve-unknown-keys + realistic-multi-server-fixture tests (Tasks 1-2) assert survival by re-decoding output. |
| T-10-04 | Tampering | Path traversal via a crafted `--root` writing outside the intended file | medium | mitigate | `filepath.Join(projectRoot, ".mcp.json"/"opencode.json")` where `projectRoot` is already `filepath.Abs`-resolved upstream in the /api/v1/init handler (workspace.go:120); Codex path is a fixed `~/.codex/config.toml` (no user-supplied path segment). No symlink-following of the target path into unexpected locations. |
| T-10-05 | Information Disclosure | World-readable config files | low | mitigate | Write all files with `0600` and create parent dirs `0700`, matching runInteractiveInit's existing `os.WriteFile(configPath, ..., 0600)` (init.go:239). |
