<objective>
Build the wizard's server-start step (D-14): start the daemon only when prerequisites pass, skip when already running, abort on a PostgreSQL FAIL, and degrade on Windows — all without newly breaking the already-broken Windows build.

Purpose: `runDoctorCmd` calls `os.Exit(1)` on any fail, so the wizard cannot reuse it and keep control. This step calls `doctor.RunAll` directly, inspects the PostgreSQL check, and gates the daemon launch. RESEARCH Pitfall 4 + Pattern 4 (VERIFIED) require the daemon call to live in a `!windows` file because `runServeDaemonFn` itself is compiled out on Windows.
Output: `init_serve.go` (build-tag-free step logic + status enum), `init_serve_unix.go` (`//go:build !windows`, real daemon launch), `init_serve_windows.go` (`//go:build windows`, manual-instruction stub), `init_serve_test.go`. No other file changes.
</objective>
