<objective>
Implement Fix A (D-06a): reorder `processFile` in `internal/watcher/watcher.go` so the content-hash dedup check runs BEFORE `extractAndUpsertEdges`. Byte-identical content produces identical edges, so re-running tree-sitter edge extraction on every unchanged file at startup is pure waste — the observed `graph edges extracted` log storm (D-05).

Purpose: Eliminate redundant graph edge re-extraction for unchanged files. This is the cheap half of the locked A+B solution and also covers the git-checkout case (mtime rewritten so the fast-path misses, content read, hash matches, edges skipped).

Output: A reordered `processFile` and a regression test proving edges are not re-extracted for byte-identical content.
</objective>
