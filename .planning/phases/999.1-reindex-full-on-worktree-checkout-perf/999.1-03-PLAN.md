<objective>
Implement the watcher wiring for Fix B (D-06b): make the mtime+size fast-path survive process restart by warming the in-memory `w.fileCache` from the DB at watcher startup, and keep the persisted fingerprint current by writing `mod_time` + `file_size` (from `os.Stat`) on every index.

Purpose: An unchanged file on a plain restart (or a newly-registered worktree) is skipped by the cheap Stage-1 fast-path with no read/hash/extraction — eliminating the serve-startup graph-extraction storm (the primary complaint). Combined with plan 01 (Fix A), the git-checkout case is also handled: checkout rewrites mtime so the warmed entry misses, the file is read, and the content-hash dedup (plan 01) then skips edge extraction.

Output: a `warmFileCacheFromDB` method wired into the startup scan path, os.Stat-sourced fingerprint persistence in `processFile`, and an integration test proving restart-skip behavior against the test DB.
</objective>
