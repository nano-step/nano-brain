# Phase 999.1 Plan 03: Watcher Fix-B Wiring Summary

Wired Fix B (D-06b) into the watcher: DB-warmed file cache on startup so unchanged files are skipped on process restart without read/hash/extraction.

## What Was Built

### Task 1 — `PreloadFileStateByWorkspace` in `WatcherQuerier` + `ModTime`/`FileSize` persistence

- Added `PreloadFileStateByWorkspace(ctx, sqlc.PreloadFileStateByWorkspaceParams) ([]sqlc.PreloadFileStateByWorkspaceRow, error)` to the `WatcherQuerier` interface (`internal/watcher/watcher.go:49`).
- Populated `ModTime: sql.NullTime{Time: info.ModTime(), Valid: true}` and `FileSize: sql.NullInt64{Int64: info.Size(), Valid: true}` in the `UpsertDocumentBySourcePathParams` literal in `processFile`, using the `info` already in scope from the earlier `os.Stat` call. No second stat added.
- Added a no-op `PreloadFileStateByWorkspace` stub to `mockQuerier` in `watcher_test.go` to keep the unit suite compiling.

Commit: `a53251d`

### Task 2 — `warmFileCacheFromDB` + `warmed` guard + `scanCollection` wiring

- Added `warmed map[string]bool` field to `Watcher` struct; initialized in `New`.
- Added `warmFileCacheFromDB(ctx, col)` method:
  - Under `w.mu`: if `w.warmed[col.dirPath]` is true, return immediately. Otherwise mark true and proceed.
  - Calls `w.queries.PreloadFileStateByWorkspace` for the collection.
  - On error: logs a warning, returns (cold-scan fallback, scan continues normally).
  - For each valid row (both `ModTime.Valid` and `FileSize.Valid`): populates `w.fileCache[row.SourcePath]` under `w.fileCacheMu`, skipping entries already present.
  - Logs a debug line with the count warmed.
- Wired `w.warmFileCacheFromDB(ctx, col)` at the top of `scanCollection`, before `filepath.WalkDir`. Because it is idempotent, safe to call on every scan tick.

Commit: `3fc2f50`

### Task 3 — Integration test `TestWarmFileCache`

Created `internal/watcher/watcher_integration_test.go` (`//go:build integration`, package `watcher`):

1. Sets up isolated test schema via `testutil.SetupTestDB(t)`.
2. Creates a temp dir + file; registers workspace via `q.UpsertWorkspace`.
3. Watcher A runs `scanCollection` → asserts document row has non-null `mod_time` and `file_size`.
4. Captures `updated_at` before restart simulation.
5. Watcher B (fresh `New(...)`, same `db`+`q`) runs `scanCollection` → asserts `fileCache` contains the file (preload ran) and `updated_at` is unchanged (file was skipped).
6. Modifies the file (different content + size); watcher B re-scans → asserts `updated_at` advances (changed file re-indexed).

Test result: `PASS` in 0.28s against `nanobrain_test` (migration 00029 applied cleanly).
