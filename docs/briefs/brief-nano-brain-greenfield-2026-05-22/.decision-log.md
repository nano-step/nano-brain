### Rationale

Go was chosen over the three other candidates (Rust, TypeScript, Hybrid) on the following grounds:

**vs. Rust:** Rust provides stronger compile-time correctness guarantees (data races impossible by construction, sqlx compile-time SQL). However, the solo-developer + AI agent constraint is decisive: Rust's 3–6 month borrow checker learning curve, multi-minute full rebuilds, and AI agent friction (borrow checker workarounds, async lifetime errors) create an unsustainable development tax for one person. One production team documented a 6-week rewrite estimated at 4 weeks, with the delta coming from async borrow checker errors. Go's runtime race detection (`go test -race`) combined with an architecture that keeps all shared mutable state in PostgreSQL (not in-process) achieves an acceptable correctness posture at lower operational cost.

**vs. TypeScript:** TypeScript has no concurrency safety model and its type system is escapable (`any`). Schema drift is runtime-only. These properties invert the reliability north star. TypeScript's development velocity advantage is real but not the selection criterion.

**vs. Hybrid (Rust+TypeScript):** Two build systems, two test suites, two sets of dependencies, and an FFI boundary maintained by a team of one. Prisma's hybrid pattern shows 18.4ms IPC overhead and significant operational complexity. The correctness gain from the Rust core does not justify the cost at this project's scale.

### Selected stack components

| Component | Choice |
|---|---|
| Language | Go 1.23+ |
| PostgreSQL driver | pgx v5 + pgvector-go |
| SQL compile-time checking | sqlc |
| HTTP server | net/http stdlib or Echo |
| MCP protocol | Official MCP Go SDK (github.com/modelcontextprotocol/go-sdk) |
| Migrations | goose |
| Integration testing | testcontainers-go |
| File watching | fsnotify |

### Key risk mitigations accepted

- **Runtime race detection**: `go test -race` runs in CI on every push. Architecture minimizes in-process shared mutable state; pgxpool is goroutine-safe.
- **Schema drift**: sqlc generates type-safe Go from SQL at build time; schema drift fails the build, not production.
- **Docker image**: Go produces a ~15MB scratch-based binary with no runtime dependencies.
