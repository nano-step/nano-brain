Installation

nano-brain requires Node.js 18+. Install globally or use via npx.

# Global install (recommended)
npm install -g nano-brain

# Or use without installing
npx nano-brain --version

Initialize a workspace

Run nano-brain init from your project root. This creates ~/.nano-brain/config.yml, indexes your codebase, and generates embeddings.

# Run from your project directory
cd /your/project
nano-brain init
What init does: Creates config, scans codebase with Tree-sitter, extracts symbols, generates embeddings, and starts a local Qdrant instance (via Docker if available).

After init, check everything is healthy:

nano-brain status

Connect to your AI agent

nano-brain speaks MCP (Model Context Protocol) via stdio, HTTP, or SSE. Add it to your agent's config:

OpenCode

# ~/.config/opencode/opencode.json
{
  "mcp": {
    "nano-brain": {
      "type": "stdio",
      "command": "nano-brain",
      "args": ["mcp"]
    }
  }
}

Claude Code

# ~/.claude.json
{
  "mcpServers": {
    "nano-brain": {
      "command": "nano-brain",
      "args": ["mcp"]
    }
  }
}

HTTP / SSE (Docker or remote)

# Start HTTP server
nano-brain mcp --http --port 8282

# Connect via SSE
nano-brain mcp --http --port 8282
# MCP endpoint: http://localhost:8282/mcp
# SSE endpoint:  http://localhost:8282/sse

Configuration — Collections

Collections define which directories nano-brain indexes. Each collection is a named directory with a glob pattern.

# ~/.nano-brain/config.yml
collections:
  memory:
    path: ~/.nano-brain/memory
    pattern: "**/*.md"
    update: auto
  sessions:
    path: ~/.nano-brain/sessions
    pattern: "**/*.md"
    update: auto
  docs:
    path: ~/projects/my-app/docs
    pattern: "**/*.md"
    update: auto
KeyDefaultDescription
path—Absolute or ~ path to the directory
pattern**/*.mdGlob pattern for files to index
updateautoauto (watch) or manual

Configuration — Workspaces

Workspaces define which project roots are allowed to create databases. Since v2026.8.19, paths not in this list fall back to the nearest configured workspace instead of creating orphaned databases.

workspaces:
  /Users/you/projects/my-app:
    codebase:
      enabled: true
      maxFileSize: 300KB
      extensions:
        - .ts
        - .tsx
        - .py
      exclude:
        - "**/node_modules/**"
        - "**/dist/**"
  /Users/you/projects/api:
    codebase:
      enabled: true
Backward compatible: If workspaces: is absent or empty, any --root path creates a database (original behavior preserved).

Configuration — Vector store

nano-brain uses Qdrant as the vector store. Run it locally via Docker or point to a remote instance.

vector:
  provider: qdrant
  url: http://localhost:6333
  collection: nano-brain-768  # must match dimensions
# Start Qdrant via nano-brain Docker helper
nano-brain docker start

# Or run directly
docker run -d -p 6333:6333 qdrant/qdrant

Configuration — Codebase indexing

codebase:
  enabled: true
  maxFileSize: 300KB
  extensions:
    - .ts
    - .tsx
    - .js
    - .py
  exclude:
    - "**/node_modules/**"
    - "**/dist/**"
    - "**/.git/**"

Codebase indexing uses Tree-sitter to extract symbols, build call graphs, and map file dependencies. Works for TypeScript, JavaScript, and Python.

MCP Tools — Search & Retrieval

memory_searchBM25 keyword search. Fast, exact matching via SQLite FTS5.
memory_vsearchSemantic vector search via Qdrant embeddings.
memory_queryFull hybrid search — BM25 + vector + RRF + PageRank + reranking.
memory_getRetrieve a document by path or docid (e.g. #abc123).
memory_multi_getBatch retrieve documents by glob pattern.

MCP Tools — Memory Management

memory_writeWrite an observation to the daily log. Supports tags and supersedes.
memory_statusIndex health, collection counts, model status, graph stats.
memory_updateTrigger a full reindex of all collections.
memory_tagsList all tags with document counts.
memory_consolidateTrigger LLM-based memory consolidation manually.
memory_consolidation_statusView consolidation queue stats and recent logs.
memory_importanceView document importance scores.
memory_suggestionsProactive next-query predictions based on patterns.
memory_wake_upLoad session context on agent startup.

MCP Tools — Code Intelligence

code_context360° view of a code symbol: callers, callees, flows, centrality, cluster.
code_impactChange impact analysis — upstream/downstream BFS traversal.
code_detect_changesMap a git diff to affected symbols (staged/unstaged/all).
memory_index_codebaseIndex codebase files in the current workspace using Tree-sitter AST.

MCP Tools — Graph & Intelligence

memory_focusFile dependency context: imports, exports, centrality, cluster.
memory_graph_statsDependency graph overview: files, symbols, edges, cycles.
memory_symbolsCross-repo symbol query (Redis keys, MySQL tables, API endpoints).
memory_impactCross-repo impact analysis: writers vs. readers of a symbol.
memory_graph_queryBFS traversal from an entity through the knowledge graph.
memory_relatedFind related memories via entity graph connections.
memory_timelineTemporal view of entity changes over time.
memory_expandExpand context around a retrieved document.
memory_traverseGraph traversal with depth and filter controls.

CLI — 27 commands

All commands accept --db=<path> and --config=<path> global flags.

Server

nano-brain mcpStart MCP server (stdio)
nano-brain mcp --http --port 8282Start MCP server (HTTP/SSE)
nano-brain mcp --daemonStart as background daemon

Index & Search

nano-brain initInitialize workspace
nano-brain statusShow index health and stats
nano-brain query "<text>"Hybrid search
nano-brain search "<text>"BM25 search
nano-brain vsearch "<text>"Vector search
nano-brain embedGenerate embeddings for un-embedded chunks
nano-brain context <symbol>Code symbol context

Collections & Workspace

nano-brain collection listList all collections
nano-brain collection add <name> <path>Add a collection
nano-brain collection remove <name>Remove a collection
nano-brain rm <path>Remove a document or workspace

Docker

nano-brain docker startStart nano-brain + Qdrant containers
nano-brain docker statusContainer health status
nano-brain docker stopStop containers
