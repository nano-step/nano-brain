Core capabilities

Everything your AI agent needs
to remember

Six subsystems working together to give your agent persistent, searchable, evolving memory.

∑
Hybrid Search Pipeline
6-stage ranking pipeline: BM25 via SQLite FTS5, vector cosine via Qdrant, RRF fusion (k=60), PageRank centrality boost, supersede demotion, and VoyageAI neural reranking.
BM25 Qdrant RRF PageRank Reranking
⊕
Code Intelligence
Tree-sitter AST parsing for TypeScript, JavaScript, and Python. Extracts symbols, builds call graphs (caller→callee), file dependency graphs, and flow detection for impact analysis.
Tree-sitter Call Graph Impact Analysis
⬡
MCP Server — 22+ Tools
Full MCP server with stdio, HTTP, and SSE transports. Tools for search, memory management, code intelligence, graph traversal, indexing, and workspace control.
stdio HTTP SSE
◈
Memory Intelligence v2
Three adaptive layers: LLM-based entity extraction and categorization, Thompson Sampling bandits that tune search weights from telemetry, and preference learning that adapts to retrieval patterns.
Entity Graph Bandits Preference Learning
⟳
Session Harvesting
Automatically ingests AI session files from OpenCode and Claude Code. Converts JSON sessions into markdown observations, indexes them with project-hash workspace isolation.
OpenCode Claude Code Auto-index
▣
Workspace Isolation
Every project gets its own SQLite database keyed by workspace path hash. Cross-workspace search is opt-in. Config guards prevent orphaned databases from accumulating on disk.

Architecture

11-stage search pipeline

Every query passes through up to 11 ranked stages before returning results.

BM25 / FTS5
→
Vector (Qdrant)
→
RRF Fusion
→
PageRank Boost
→
Supersede Filter
→
Neural Rerank

Search parameters are tuned automatically via Thompson Sampling bandits based on retrieval telemetry.

Quick start

Up in 60 seconds

Install, initialize, and connect to your AI agent.

npm
OpenCode
Claude Code
# Install globally
npm install -g nano-brain

# Initialize in your workspace
npx nano-brain init

# Start MCP server (stdio)
npx nano-brain mcp

# Or HTTP mode for containers
npx nano-brain mcp --http --port 8282
# ~/.config/opencode/opencode.json
# Option A: Local (stdio)
{
  "mcp": {
    "nano-brain": {
      "type": "local",
      "command": ["npx", "nano-brain", "mcp"],
      "enabled": true
    }
  }
}

# Option B: Docker/remote (recommended for teams)
{
  "mcp": {
    "nano-brain": {
      "type": "remote",
      "url": "http://localhost:3100/mcp",
      "enabled": true
    }
  }
}
# .mcp.json (project root) — via SSE to running server
{
  "mcpServers": {
    "nano-brain": {
      "command": "npx",
      "args": ["mcp-remote", "http://localhost:3100/sse"]
    }
  }
}

# Or start Docker server first:
$ npx nano-brain docker start

MCP Tools

30 tools across 4 categories

Every tool is accessible via stdio, HTTP, or SSE transport.

Search & Retrieval

memory_search
BM25 keyword
BM25 full-text search via SQLite FTS5. Fast, exact-match keyword lookup across all indexed documents.
memory_vsearch
Semantic vector
Semantic vector search via Qdrant. Finds documents by meaning, not just keywords.
memory_query
Hybrid search
Full hybrid search: BM25 + vector + RRF fusion + 11 ranking stages. Best results, slightly slower.
memory_get
By path / docid
Retrieve a single document by file path or short docid (e.g. #abc123).
memory_multi_get
Batch / glob
Batch-retrieve multiple documents matching a glob pattern.

Memory Management

memory_write
Write to daily log
Write an observation to the daily log. Supports --tags and --supersedes for replacing stale entries.
memory_status
Index health
Show index health: document counts, embedding status, graph stats, model info.
memory_update
Trigger reindex
Trigger a full reindex of all configured collections.
memory_tags
List all tags
List all tags in the index with document counts per tag.
memory_consolidate
LLM consolidation
Trigger LLM-based memory consolidation — merges, deduplicates, and distills entries.
memory_suggestions
Proactive next queries
Returns proactive next-query predictions based on detected retrieval patterns.

Code Intelligence

code_context
Symbol 360° view
360° view of a code symbol: callers, callees, call flows, PageRank centrality, cluster.
code_impact
Change impact BFS
BFS traversal to find all upstream and downstream symbols affected by a change.
code_detect_changes
Git diff → symbols
Maps a git diff (staged/unstaged/all) to the set of affected symbols.
memory_index_codebase
AST index workspace
Index codebase files in the current workspace using Tree-sitter AST parsing.

Graph & Intelligence

memory_focus
File dependency
File dependency context: what this file imports, what imports it, centrality score.
memory_graph_stats
Graph overview
Dependency graph overview: total files, symbols, edges, detected cycles.
memory_graph_query
BFS traversal
BFS traversal from an entity node through the knowledge graph.
memory_connect
Connect entities
Create a relationship edge between two entities in the knowledge graph.
memory_connections
List connections
List all relationship edges for a given entity.
memory_related
Related via graph
Find memories related to a topic via entity graph connections.
memory_timeline
Temporal view
Temporal view of how an entity or topic has changed over time.
memory_traverse
Graph traversal
Configurable graph traversal with depth limit and edge-type filters.
memory_expand
Expand context
Expand context around a retrieved document (surrounding chunks, linked entities).
memory_symbols
Cross-repo symbols
Cross-repo symbol query — find Redis keys, MySQL tables, API endpoints, queues.
memory_impact
Cross-repo impact
Cross-repo impact analysis — find all writers and readers of a symbol.
memory_importance
Importance scores
View importance scores for documents (based on retrieval frequency and recency).
memory_learning_status
Learning stats
View learning system status: telemetry record count, bandit variants, config version.
memory_wake_up
Session context
Load session context on agent startup — returns recent memories and active focus.
