The problem
AI agents forget everything when the session ends. Every new conversation starts from zero. Your team re-explains architecture, re-discovers patterns, re-states decisions that were made last week.
  • Self-hosted. Your data stays on your server. No cloud dependency, no vendor lock-in.
  • Works everywhere. OpenCode, Claude Code, Cursor, any MCP client. Not tied to one editor.
  • Actually production-ready. Not a toy demo. Go binary, PostgreSQL, 16 MCP tools, hybrid search, and code intelligence.
  • The only tool with code intelligence. Symbol graphs, call chains, impact analysis. No competitor offers this.
What it does

Three capabilities, one binary.

nano-brain combines memory, search, and code intelligence into a single self-hosted server your AI agent talks to via MCP.

Session Memory

Auto-ingest from OpenCode and Claude Code sessions. Decisions, patterns, and code context persist across sessions, machines, and team members.

Hybrid Search

BM25 full-text + pgvector HNSW cosine similarity + Reciprocal Rank Fusion. Find anything by keyword or meaning, with recency weighting.

Code Intelligence

Symbol graphs, call chain tracing, impact analysis, and flow visualization. Ask "what breaks if I change this?" and get real answers.

When to use it

Built for real development workflows.

Multi-machine

Work on any device, keep all context

Deploy nano-brain on a VPS. Work on office PC, home laptop, personal machine. Every session gets harvested. Switch machines, pick up where you left off.

One server, whole team

Every developer's AI agent connects to the same PostgreSQL. Decisions, architecture notes, code intelligence. New hires get full context from day one.

Code archaeology

Pre-commit safety

Run impact analysis on changed files. Discover what depends on them. Prevent regressions before they reach your CI pipeline.

Get started

Running in under 5 minutes.

1

Install

npm install -g @nano-step/nano-brain

Or build from source: CGO_ENABLED=0 go build -o nano-brain ./cmd/nano-brain

2

Start server

nano-brain serve -d

Requires PostgreSQL 17 + pgvector. Run nano-brain doctor to check prerequisites.

3

Connect your agent

Add to your MCP config:
{"mcp":{"nano-brain":{
  "type":"http",
  "url":"http://localhost:3100/mcp"
}}}

Language support

Ruby / Rails — shipped.

Full code intelligence for Ruby on Rails applications. Routes, control-flow graphs, cross-file resolution.

Rails Routes

Extracts resources, HTTP verbs, namespace, scope, mount. Full route-to-controller mapping.

Control-Flow Graphs

if/else, loops, begin/rescue, method definitions. Branch-aware edges for accurate tracing.

Cross-File Resolution

Class-to-file index, resolver, reconcile edges. Flows reach 20-34 nodes across controller, service, and model layers.
