Features

Everything an AI agent needs to remember across sessions.

Hybrid Search

BM25 + pgvector + RRF fusion + recency decay. Find anything by keyword or meaning.

9 MCP Tools

query, search, vsearch, get, write, tags, status, update, wake_up. Works with any MCP client.

Session Harvesting

Auto-ingest OpenCode and Claude Code sessions. Never lose context.

File Watcher

fsnotify directory monitoring with debounce. Documents stay in sync.

Content Addressing

SHA-256 deduplication. Write the same thing twice, stored once.

Config Hot-Reload

POST /api/reload-config. Change search params without restart.

Single Binary

CGO_ENABLED=0 Go binary. No runtime dependencies.

V1 Migration

Quick Start

Three steps to persistent AI memory.

1. Build

CGO_ENABLED=0 go build -o nano-brain ./cmd/nano-brain

2. Start PostgreSQL

docker run -d --name nanobrain-pg -p 5432:5432 \
  -e POSTGRES_USER=nanobrain \
  -e POSTGRES_PASSWORD=nanobrain \
  -e POSTGRES_DB=nanobrain_dev \
  pgvector/pgvector:pg17

3. Run

DATABASE_URL="postgres://nanobrain:nanobrain@localhost:5432/nanobrain_dev" \
  ./nano-brain

Then register a workspace, write, and search:

# Register workspace
curl -X POST http://localhost:3100/api/v1/init \
  -H "Content-Type: application/json" \
  -d '{"root_path":"/path/to/project","name":"my-project"}'

# Write a document
curl -X POST http://localhost:3100/api/v1/write \
  -H "Content-Type: application/json" \
  -d '{"workspace":"<hash>","source_path":"notes/decision.md",
       "content":"# Decision\nUse PostgreSQL.","tags":["decision"]}'

# Hybrid search
curl -X POST http://localhost:3100/api/v1/query \
  -H "Content-Type: application/json" \
  -d '{"workspace":"<hash>","query":"database decision"}'
