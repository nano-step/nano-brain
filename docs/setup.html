Recommended setup

Get nano-brain running

Docker + Ollama on the host (for GPU). Swap highlighted values for your project.

Docker required Ollama on host (recommended) macOS · Linux

1 Prerequisites

  • Docker Desktop running (install)
  • Node.js 18+ (node --version)
  • Ollama running on the host — for GPU embedding
  • Embedding model pulled: nomic-embed-text

Install Ollama + pull embedding model

$ terminal 
# macOS
brew install ollama

# Start with GPU support + keep model in memory
OLLAMA_KEEP_ALIVE=30m OLLAMA_NUM_PARALLEL=4 ollama serve

# Pull the embedding model (768-dim, fast)
ollama pull nomic-embed-text
Why Ollama on the host? Running Ollama natively uses Metal GPU on Apple Silicon → ~50–200ms per embed. Inside Docker = ~15s per embed. nano-brain talks to it via host.docker.internal:11434.

2 Create config file

Create ~/.nano-brain/config.yml. Replace the highlighted values with your project info.

📄 ~/.nano-brain/config.yml 
logging:
  enabled: true
  level: info

collections:
  memory:
    path: ~/.nano-brain/memory
    pattern: "**/*.md"
    update: auto
  sessions:
    path: ~/.nano-brain/sessions
    pattern: "**/*.md"
    update: auto

embedding:
  provider: ollama
  url: http://host.docker.internal:11434
  model: nomic-embed-text
  dimensions: 768
  maxChars: 16000
  rpmLimit: 100

vector:
  provider: qdrant
  url: http://qdrant:6333          # internal Docker network
  collection: nano-brain-768

workspaces:
  /path/to/your/project:       # ← change this
    codebase:
      enabled: true
      maxFileSize: 300KB
      extensions:
        - .ts
        - .tsx
        - .js
        - .py
      exclude:
        - "**/node_modules/**"
        - "**/dist/**"
        - "**/.git/**"

harvester:
  opencode:
    enabled: true
  claudeCode:
    enabled: true               # set false if not using Claude Code

reranker:
  provider: cohere             # optional — remove if no API key
  model: rerank-v3.5
  apiKey: YOUR_COHERE_KEY        # ← or remove reranker section entirely
Change before starting: Replace /path/to/your/project with the absolute path to your project (e.g. /Users/you/projects/my-app). This controls which workspace is indexed and which DB is created.
No Cohere key? Remove the reranker: section entirely. nano-brain runs fine without it — reranking is optional. Search still uses BM25 + vector + 9 other ranking stages.

3 Start with Docker

The docker command handles everything — starts Qdrant + nano-brain server, waits for health check, migrates config if needed.

$ terminal 
# Start nano-brain + Qdrant
npx nano-brain docker start

# Check status
npx nano-brain docker status

# View logs if something's wrong
docker logs nano-brain-server -f

# Stop
npx nano-brain docker stop
What docker start does: Copies config.default.yml → ~/.nano-brain/config.yml if not present, creates data/memory/sessions/logs dirs, runs docker compose up -d, polls http://localhost:3100/health until ready (up to 20 retries × 3s).

docker-compose.yml environment variables

Set these before running docker start to override defaults:

📄 .env (or export in shell) 
# Path to your workspace (mounted read-only into container)
NANO_BRAIN_WORKSPACE=/path/to/your/project

# nano-brain home dir (default: ~/.nano-brain)
NANO_BRAIN_HOME=~/.nano-brain

# OpenCode session storage (default: ~/.local/share/opencode)
OPENCODE_DATA_DIR=~/.local/share/opencode
Important: NANO_BRAIN_WORKSPACE must match the path in config.yml → workspaces:. If they differ, nano-brain will fall back to the first configured workspace and log a warning.

4 Connect MCP agent

Once the server is running at http://localhost:3100, connect your AI agent:

OpenCode
Claude Code
📄 ~/.config/opencode/opencode.json 
{
  "mcp": {
    "nano-brain": {
      "type": "remote",
      "url": "http://localhost:3100/mcp",
      "enabled": true
    }
  }
}

Restart OpenCode after editing this file. Use "type": "local" with "command": ["npx", "nano-brain", "mcp"] for stdio mode without Docker.

📄 .mcp.json (project root) 
{
  "mcpServers": {
    "nano-brain": {
      "command": "npx",
      "args": ["mcp-remote", "http://localhost:3100/sse"]
    }
  }
}

Claude Code connects via SSE. The server must be running before starting Claude Code.

5 Verify it's working

$ terminal 
# Check health endpoint
curl http://localhost:3100/health

# Expected: {"status":"ok","ready":true}

# Check index status
npx nano-brain status

# Run a test search
npx nano-brain query "your search query"
First startup takes 1–3 minutes. nano-brain installs deps inside Docker, rebuilds native modules for Linux, then starts the server. Subsequent restarts are ~10s.

Troubleshooting

$ diagnostics 
# Container not starting?
docker logs nano-brain-server

# Qdrant not connecting?
curl http://localhost:6333/healthz

# Ollama not reachable?
curl http://localhost:11434/api/tags

# Check all containers
docker ps | grep nano-brain

Container setup with ai-sandbox-wrapper

This is the recommended production setup. All AI tools — OpenCode, nano-brain, and others — run inside secure Docker containers via ai-sandbox-wrapper. Your SSH keys and system files stay protected on the host.

How it works: ai-run launches a container from the ai-sandbox:latest image. nano-brain is pre-installed inside. The container mounts your workspace read-only and runs OpenCode (or any AI tool) with nano-brain available as an MCP server via stdio.

Install ai-sandbox-wrapper

$ terminal
# Install globally
npm install -g @kokorolx/ai-sandbox-wrapper

# Or run directly
npx @kokorolx/ai-sandbox-wrapper

Run OpenCode with nano-brain inside the sandbox

$ terminal
# Start OpenCode in sandbox — nano-brain is pre-installed
ai-run opencode

# Run nano-brain commands directly via ai-run
ai-run npx nano-brain status
ai-run npx nano-brain query "your search"

# Debug mode — show suppressed tree-sitter warnings
AI_RUN_DEBUG=1 ai-run npx nano-brain query "hello"
Auto-repair built in: When nano-brain detects a native module failure (e.g. tree-sitter compiled for wrong arch), ai-run automatically rebuilds the bindings and retries once. Non-fatal symbol-graph warnings are suppressed by default — use AI_RUN_DEBUG=1 to see them.

nano-brain MCP config inside the container

OpenCode reads MCP config from ~/.config/opencode/opencode.json (mounted from host). Add nano-brain in stdio local mode:

📄 ~/.config/opencode/opencode.json
{
  "mcp": {
    "nano-brain": {
      "type": "local",
      "command": ["npx", "nano-brain", "mcp"],
      "enabled": true
    }
  }
}

Or, if running from source (e.g. development):

📄 ~/.config/opencode/opencode.json — from source
{
  "mcp": {
    "nano-brain": {
      "type": "local",
      "command": ["bun", "run", "/path/to/nano-brain/src/index.ts", "mcp"],
      "enabled": true
    }
  }
}

Workspace config inside the container

The container mounts your workspace at the same absolute path as the host. Your ~/.nano-brain/config.yml (mounted from host) should reference that same path:

📄 ~/.nano-brain/config.yml
workspaces:
  /Users/you/projects/my-app:    # same path as host
    codebase:
      enabled: true

# Sessions from OpenCode are harvested automatically
harvester:
  opencode:
    enabled: true
  claudeCode:
    enabled: true

Available tools in ai-sandbox:latest

opencode nano-brain claude gemini aider codex kilo amp bun npx

Alternative: Local stdio (no Docker)

If you don't want Docker, run nano-brain directly as a local MCP server. You still need Qdrant — easiest via docker run for just Qdrant.

$ terminal
# Install globally
npm install -g nano-brain

# Start just Qdrant
docker run -d -p 6333:6333 qdrant/qdrant:latest

# Update config.yml vector URL for local Qdrant
# vector.url: http://localhost:6333

# Initialize workspace
cd /path/to/your/project
npx nano-brain init

# Start MCP server (stdio — for OpenCode/Claude Code)
npx nano-brain mcp
📄 OpenCode config (stdio mode)
{
  "mcp": {
    "nano-brain": {
      "type": "local",
      "command": ["npx", "nano-brain", "mcp"],
      "enabled": true
    }
  }
}
Local mode caveat: In stdio mode, nano-brain starts a new process per agent session. Docker mode keeps one persistent server process — better for performance and shared state across multiple agent sessions.

Claude Code harvesting

nano-brain can automatically harvest Claude Code sessions. Enable in config:

📄 ~/.nano-brain/config.yml — harvester section
harvester:
  claudeCode:
    enabled: true
    sessionDir: ""   # default: ~/.claude/projects
                     # override if your sessions are elsewhere

Sessions are auto-indexed every 2 minutes. After enabling, run npx nano-brain status to confirm sessions are being picked up.

Session format: Claude Code stores sessions as JSONL in ~/.claude/projects/. nano-brain's ClaudeCodeAdapter converts them to markdown observations and indexes them with your workspace's project hash.
