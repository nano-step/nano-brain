## Product Contract

A standalone `internal/links/` package provides:

- `Parse(content string) []Link` — extract `[[wikilinks]]` from any UTF-8 content
- `Resolver` with a workspace-scoped, case-insensitive title cache + ID existence check
- `Extractor.Extract(ctx, doc)` — diff-and-upsert `references` edges in a single transaction, scoped to memory + session-summary collections

A forward-only migration `00011_graph_edge_references.sql` extends `graph_edges.edge_type` CHECK to accept `'references'`. Down rollback is documented as fail-if-references-exist; operator removes the rows first.

Wiring: `handlers/write`, `handlers/reindex`, `harvest/harvest` call `resolver.FlushWorkspace(wh)` THEN `extractor.Extract(ctx, doc)` after each successful memory/summary write. Extractor errors are logged-warn, not failed-write (best-effort).

## Acceptance Criteria

1. **Migration 00011** exists, applies cleanly, allows `'references'` edges, Down fails-with-references then succeeds-without (documented procedure).
2. **Zero internal deps in production files**: `grep '"github.com/nano-brain/nano-brain/internal/' internal/links/*.go | grep -v _test.go` is empty.
3. **Parser** correctly handles 8 cases (simple ID, simple title, mixed, escaped, malformed open, nested brackets, >200-char content, unicode titles).
4. **Resolver** correctly handles 6 cases (ID hit/miss, case-insensitive title, ambiguous, cache hit, flush).
5. **Extractor** integration test: write A → write B with `[[A]]` → Extract(B) → 1 `references` row in graph_edges. Re-extract → still 1 row. Add C with `[[A]]` → 2 rows.
6. **Cache-flush ordering**: rename A's title; subsequent write with new title resolves correctly.
7. **No regression**: `go test -race -short ./...` passes including existing `internal/graph/` tests (callers ignore unknown edge types).
8. `go build ./...` exit 0, `go vet ./internal/links/...` clean.

## Design Notes

- **Commands**: Parse (pure), Resolve (cached DB query), Extract (transactional upsert + bus publish).
- **Queries**: 5 new sqlc queries (see issue body).
- **API**: no HTTP surface in this story (Story 9.4 adds endpoints that consume the extractor's output).
- **Tables**: 1 modified (`graph_edges` CHECK extended). No new tables.
- **Domain rules**:
  - Wikilink content: `[^\[\]\n]{1,200}` inside `[[...]]` — single line, max 200 chars, no nesting.
  - Title match: case-insensitive, full-string equality (NOT substring).
  - Ambiguous title (>1 match): extractor inserts edge to most-recent (by `updated_at`), stamps `metadata.ambiguous = true` + candidate list.
  - Cross-workspace edges: extractor refuses (defense in depth).
  - Scope: extraction runs only when `doc.collection IN ('memory', 'session-summary:%')`. Source-code documents skip.
- **UI surfaces**: none in this story.
