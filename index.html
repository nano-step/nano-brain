Features

Premium local memory with a QMD-grade hybrid pipeline, SQLite foundations, codebase indexing, and MCP-first ergonomics.

🔍

Hybrid Search Pipeline

Query expansion → parallel BM25 + vector search → RRF fusion → LLM reranking

🧠

Local AI Models

nomic-embed-text-v1.5, bge-reranker-v2-m3, qmd-query-expansion-1.7B — all GGUF, fully offline

🗄️

SQLite + sqlite-vec

FTS5 for BM25 keyword search, sqlite-vec for vector similarity — single-file database

🔌

MCP Server

10 tools via stdio transport, integrates with any MCP-compatible AI agent

📚

Multi-Collection

Sessions, daily logs, curated memory — all indexed and searchable

🔒

Privacy First

100% local processing, no cloud dependencies, your data stays on your machine

🧬

Codebase Indexing

Index source files with structural boundary detection. Workspace-scoped search across sessions and code.

API Reference

MCP tools for hybrid search, retrieval, and maintenance.

memory_search

BM25 keyword search.

query (required) limit (default 10) collection workspace

memory_vsearch

Semantic vector search using embeddings.

query (required) limit (default 10) collection workspace

memory_query

Full hybrid search with query expansion, RRF fusion, and LLM reranking.

query (required) limit (default 10) collection minScore workspace

memory_get

Retrieve document by path or docid (#abc123).

id (required) fromLine maxLines

memory_multi_get

Batch retrieve by glob pattern.

pattern (required) maxBytes (default 50000)

memory_write

Write to daily log or MEMORY.md.

content (required) target ("daily" or "memory")

memory_status

Show index health, collections, model status.

no params

memory_update

Trigger immediate reindex of all collections.

no params

memory_index_codebase

Index source files in the current workspace.

root (optional)

Configuration

Drop this MCP config into your OpenCode settings.

{
  "mcp": {
    "nano-brain": {
      "type": "local",
      "command": ["node", "path/to/nano-brain/bin/cli.js", "mcp"],
      "enabled": true
    }
  }
}
# ~/.config/nano-brain/config.yml
collections:
  memory:
    path: ~/.nano-brain
    pattern: "**/*.md"
    update: auto
  sessions:
    path: ~/.local/share/opencode/sessions
    pattern: "**/*.json"
    update: auto

Quick Start

CLI workflows for MCP, search, and indexing.

# MCP server
nano-brain mcp              # Start MCP server (stdio)
nano-brain mcp --http       # Start MCP server (HTTP)

# Search
nano-brain search "query"   # BM25 search
nano-brain query "query"    # Hybrid search

# Index management
nano-brain status           # Show index health
nano-brain update           # Reindex all collections
