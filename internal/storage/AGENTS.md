# storage — AGENTS.md

PostgreSQL storage layer — pool management, sqlc-generated queries, goose migrations.

## Directory Structure

```
pool.go      — NewPool (pgxpool, MaxConns=10, HealthCheckPeriod=30s, masked DSN logging)
storage.go   — package doc; package-level declarations
workspace.go — WorkspaceHash: SHA-256 of absolute root path → hex string
migrate.go   — RunMigrations: goose v3 via embedded migrations.FS
queries/     — raw SQL (INPUT to sqlc): chunks, collections, documents, embeddings, graph,
               search, telemetry, wakeup, workspaces
sqlc/        — generated Go (DO NOT EDIT)
```

## Critical Rule

**Never edit files in `sqlc/` directly.** They're fully regenerated by `sqlc generate`.
Any manual edit will be overwritten on the next run.

## Adding a New Query

1. Add SQL to the right file in `queries/*.sql` with sqlc annotations:
   ```sql
   -- name: GetDocumentByPath :one
   SELECT * FROM documents WHERE workspace_id = $1 AND source_path = $2;
   ```
2. Run `sqlc generate` from the repo root.
3. Use the new method on `*sqlc.Queries` in your handler or service.

`sqlc.yaml` wires `queries/` → `migrations/` (schema) → `internal/storage/sqlc/` (output).
The `vector` db type maps to `pgvector.Vector`.

## Migration Workflow

Migrations live in `migrations/` (embedded via `migrations.FS`).
`RunMigrations` wraps goose v3, called automatically at server startup.

Run manually:

```bash
./nano-brain db:migrate
```

Goose tracks versions in `goose_db_version`. Re-runs are idempotent.

## Integration Tests

`testutil.SetupTestDB(t)` creates an isolated PostgreSQL schema per test.

```bash
go test -race -tags=integration ./internal/storage/...
```
