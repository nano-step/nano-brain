# Spec: Health/Status workspace_count

## MODIFIED Requirements

### Requirement: Status reports actual workspace count
The `GET /api/status` and `GET /health` endpoints MUST report the real number of registered workspaces from the database.

#### Scenario: One workspace registered
- Given exactly one workspace exists in the `workspaces` table
- When a client calls `GET /api/status`
- Then the response JSON `workspace_count` field equals `1`

#### Scenario: Zero workspaces registered
- Given the `workspaces` table is empty
- When a client calls `GET /api/status`
- Then the response JSON `workspace_count` field equals `0`

#### Scenario: Many workspaces registered
- Given five workspaces exist in the `workspaces` table
- When a client calls `GET /api/status`
- Then the response JSON `workspace_count` field equals `5`

#### Scenario: Same count appears in `/health` response
- Given any number of workspaces registered
- When a client calls `GET /health`
- Then the response JSON `workspace_count` field equals the same value reported by `/api/status`

### Requirement: Health endpoint soft-fails on count query error
The health and status endpoints MUST NOT return 5xx when the workspace-count query fails.

#### Scenario: Database count query errors
- Given the database query for workspace count returns an error
- When `GET /api/status` is called
- Then the response status is 200
- And `workspace_count` is reported as `0`
- And the server logs a warning about the failed count

#### Scenario: Database unreachable (Ping fails)
- Given `pool.Ping` fails (DB unreachable)
- When `GET /health` is called
- Then the response is the existing degraded payload (`status: degraded`, `ready: false`, `reason: database unreachable`)
- And `workspace_count` field is absent or zero (no separate query attempt â€” preserves existing behavior)

## ADDED Requirements

### Requirement: CountWorkspaces sqlc query
The storage layer MUST provide a `CountWorkspaces(ctx) (int64, error)` function generated by sqlc.

#### Scenario: Query exists in generated code
- Given the source query `-- name: CountWorkspaces :one SELECT COUNT(*) FROM workspaces;` is defined in `internal/storage/queries/workspaces.sql`
- When `sqlc generate` runs
- Then `internal/storage/sqlc/workspaces.sql.go` contains a `CountWorkspaces(ctx) (int64, error)` method on `Queries`
- And the `Querier` interface includes `CountWorkspaces(ctx) (int64, error)`

### Requirement: WorkspaceCounter interface segregation
The `handlers` package MUST define a minimal `WorkspaceCounter` interface used by the `Health` handler.

#### Scenario: Interface contains only the count method
- Given the new `WorkspaceCounter` interface in `internal/server/handlers/health.go`
- Then it declares exactly one method: `CountWorkspaces(ctx context.Context) (int64, error)`
- And `*sqlc.Queries` satisfies it
- And `Health` constructor accepts a `WorkspaceCounter` parameter
