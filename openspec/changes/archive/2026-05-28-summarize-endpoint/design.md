## API Shape

```
POST /api/v1/summarize
Content-Type: application/json

{
  "workspace": "string",          // required — extracted by workspace middleware
  "source":    "opencode|claude", // optional — filter by source; "claude" matches tag "claude_code"
  "limit":     20,                // optional — max sessions to process (default 10, max 20)
  "force":     false              // optional — skip pre-check and re-run LLM even if summary exists
}

Response 200:
{
  "summarized": 12,
  "skipped":    8,
  "errors":     0
}

Response 503: summarization not configured
Response 400: invalid request
```

Note: `limit` is capped at **20** server-side (v1). This keeps requests synchronous and within HTTP timeout bounds (~10-60s/session × 20 = up to 20 min max, acceptable for manual backfill).

## Data Flow

```
POST /api/v1/summarize
        │
        ▼
workspaceMiddleware (existing) extracts workspace from JSON body
        │
        ▼
SummarizeHandler.Handle(c echo.Context)
        │  c.Bind(&req)
        │  workspace = c.Get("workspace").(string)
        │  getSummarizer() == nil → 503 "summarization not configured"
        │  clamp req.Limit: default=10, max=20
        │
        ▼
queries.ListSessionDocumentsByWorkspace(ctx, params)  // NEW sqlc query
        │  WHERE workspace_hash=$1 AND collection='sessions'
        │  AND ($2='' OR $2=ANY(tags))                // tag filter (e.g. "opencode")
        │  ORDER BY created_at DESC LIMIT $3
        │  Returns: id, title, source_path, tags, content, created_at
        │
        ▼  for each doc (serialized — no goroutines)
if !req.Force:
  GetDocumentBySourcePath(ctx, summarySourcePath(doc))
        │  summary source path = "session-summary://opencode/{sessionID}"
        │  if exists → skipped++, continue
        │
        ▼
meta := extractSummaryMeta(doc)
        │  Source:    "opencode"|"claude" (from tags: "claude_code"→"claude")
        │  SessionID: path.Base(doc.SourcePath)  e.g. "opencode://session/{id}" → id
        │  Title:     strings.TrimPrefix(doc.Title, "Session: ")
        │  CreatedAt: doc.CreatedAt
        │  (Agent, ProjectPath, Duration, ParentID: left zero — not stored in doc)
        │
        ▼
h.getSummarizer().SummarizeAndPersist(ctx, doc.Content, meta)
        │  LLM call via Pipeline → Persister.Save
        │  error → errors++, log.Warn, continue
        │  success → summarized++
        │
        ▼
return 200 SummarizeResponse{Summarized, Skipped, Errors}
```

## Key Design Decisions

**Route in `/api/v1` group**: Workspace middleware (`workspaceMiddleware`) only runs on the `data` group (`/api/v1/*`). Registering there gives workspace extraction for free — consistent with all other workspace-scoped endpoints.

**New sqlc query required**: `ListDocumentsByWorkspace` (a) doesn't return `Content` (needed for LLM), and (b) fetches all collections (memory wasteful). A new `ListSessionDocumentsByWorkspace` query with collection filter, tag filter, and LIMIT is mandatory.

**Limit cap at 20 (v1)**: Each `SummarizeAndPersist` is a synchronous LLM call (10–60s). Beyond 20, HTTP clients will timeout. Async job support deferred to a follow-up. Server clamps any request limit above 20.

**Source tag normalization**: API accepts `"opencode"` or `"claude"`. Internally, `"claude"` maps to tag `"claude_code"`. The handler translates before passing to the DB query.

**force=true semantics**: Skips the pre-check (`GetDocumentBySourcePath` for the summary doc). Always calls `SummarizeAndPersist`. The Persister hashes the **generated output** and will skip the DB write if the LLM happens to produce identical text — but the LLM call still happens. This is correct: force=true re-runs the LLM (useful when model/prompt changed), accepting the LLM cost.

**Degraded SummaryMeta**: Fields `Agent`, `ProjectPath`, `Duration`, `ParentID` are not stored on the Document record. Retroactive summarization omits them. Acceptable degradation — the LLM summary is still useful without them.

**Serialized loop (no goroutines)**: All sessions processed sequentially. Simpler code, no concurrency bugs. The v1 limit cap (20) keeps total wall-clock time manageable. Bounded concurrency via errgroup deferred to v2.

**Mutex for concurrent requests**: Add `summarizeMu sync.Mutex` to Server struct. `/api/v1/summarize` handler acquires mutex before processing. Prevents multiple concurrent summarize runs from overwhelming the LLM provider.
